## Version History Format

We use [Semantic Versioning](https://semver.org/) for documentation versions:

- **MAJOR** version: Significant restructuring or major content additions
- **MINOR** version: New sections, products, or substantial updates
- **PATCH** version: Corrections, clarifications, minor updates

### Change Categories

- **Added** - New documentation sections or features
- **Changed** - Updates to existing documentation
- **Deprecated** - Documentation for features being phased out
- **Removed** - Removed documentation sections
- **Fixed** - Corrections to errors
- **Security** - Security-related documentation updates

## Documentation Maintenance

**Update Frequency:**
- **Weekly:** Fix typos, broken links, minor clarifications
