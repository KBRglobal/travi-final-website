# ðŸ“– Glossary

> Definitions of terms used in Traviapp

---

## A

### ADR (Architecture Decision Record)
Documentation of significant architectural decisions, including context, decision, and consequences.

### API (Application Programming Interface)
The interface through which frontend and external services communicate with the backend.

### Audit Log
Immutable record of all user actions in the system for security and compliance.

---

## B

### Block (Content Block)
A structured unit of content (paragraph, heading, image, etc.) stored as JSONB.

### Body Blocks
The main content of an article, stored as an array of blocks.

---

## C

### Campaign (Newsletter)
An email sent to newsletter subscribers.

### Cluster (Topic)
A group of related RSS articles identified by AI for potential merging.

### CMS (Content Management System)
Traviapp's core functionality for managing content.

### Content Type
Category of content: article, attraction, hotel, dining, district, event, itinerary, transport.

---

## D

### DeepL
Translation API service used for 11 languages.

### Drizzle ORM
TypeScript ORM used for database operations.

### Draft
Initial status of content before publication.

---

## E

### Endpoint
A specific API URL path that handles requests.

### Entity
A database record (content, user, etc.).

---

## F

### FAQ
Frequently Asked Questions section in articles.

### Featured Image
Main image displayed with content.

### Fingerprint
Hash of content used for deduplication.

---

## G

### GPT-4o
OpenAI's model used for standard content generation.

---

## H

### Hook (React)
Reusable logic in React components (useState, useQuery, etc.).

---

## I

### i18n (Internationalization)
Multi-language support infrastructure.

### Internal Link
Link to other content within the same site.

---

## J

### JSONB
PostgreSQL JSON data type used for flexible content storage.

### JSON-LD
Structured data format for SEO.

---

## L

### Locale
Language/region code (en, ar, he, etc.).

---

## M

### Migration
Database schema change script.

### MFA/2FA
Multi-factor/Two-factor authentication using TOTP.

---

## O

### ORM (Object-Relational Mapping)
Drizzle - converts between database and TypeScript objects.

---

## P

### Personality
Writing style for AI generation (professional, friendly, etc.).

### PostgreSQL
The relational database used by Traviapp.

### PR (Pull Request)
Code change submission for review.

---

## R

### RBAC (Role-Based Access Control)
Permission system based on user roles.

### Radix UI
Accessible component library used in frontend.

### Rate Limiting
Restriction on API request frequency.

### RSS
Format for syndicating content from external sources.

---

## S

### Schema
Database table structure defined in Drizzle.

### SEO (Search Engine Optimization)
Techniques to improve search engine visibility.

### Session
User login state stored server-side.

### Slug
URL-friendly version of title.

### SPA (Single Page Application)
React application that loads once and updates dynamically.

---

## T

### TanStack Query
Library for server state management in React.

### Tailwind CSS
Utility-first CSS framework.

### Tier
AI generation quality level (standard/premium).

### TOTP
Time-based One-Time Password for 2FA.

### Translation
Content in a non-English language.

---

## U

### Upstash Redis
Cloud Redis service for caching.

---

## V

### Version (Content)
Historical snapshot of content for rollback.

### Vite
Frontend build tool.

---

## W

### Webhook
HTTP callback for event notifications.

### Workflow
Content approval process.

---

## Z

### Zod
TypeScript schema validation library.
